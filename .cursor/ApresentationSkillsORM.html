⚙️ Execution — Implementação Técnica Profunda
7 commands de Execution + 24 scripts (10 PS1 + 14 Python) + 12 rules · mantêm o pack saudável e sincronizado
/autostart
Auto-start Bootstrap
Valida espelhos (bootstrap-mirror-symlinks.ps1), detecta *.dpr/*.lpr, aciona criação interativa P1..P9 se necessário.
▸ Rule: project-autostart-bootstrap_V1.0.1
/iniciar
Iniciar sessão
Variante leve do autostart — valida ambiente e exibe estado do projeto sem criar arquivos novos.
▸ alias de /autostart para sessões recorrentes
/syncdb
Sincronizar índices FTS5
Re-indexa 3 bases SQLite — pack (.cursor/), workspace (.workspace/) e docs locais (.docs/) — com hash SHA-256 incremental.
▸ Script: pack_index_db.py V1.1.0
/sync-cursor-pack
Propagar pack entre projetos
Copia skills, agents, templates e scripts de .cursor/ do projeto fonte para projetos destino, preservando versionamento.
▸ Script: sync-cursor-pack.ps1
/consolidar
Auditoria read-only do pack
Verifica 1548 checks: versionamento de skills/rules, symlinks, nomenclatura, convenções. Gera relatório PASS/FAIL com criticidade.
▸ Script: validate_pack.py · 1548 checks · 0 CRITICAL
/migration-plan
Plano de migração documental
Analisa gaps na Documentation/, propõe fases com matriz origem/destino, dependências entre documentos e estimativas.
▸ Template: documentation-migration-plan_V1.0.md
/validate-docs
Validar documentação
Verifica estrutura, formatos, FileVersion, links internos e conformidade com as 12 rules do pack .cursor/.
▸ Script: check_docs_version · rule: documentation-file-versioning
