# Exemplo — hub de documentação (`Documentation/`)

Este ficheiro é um **exemplo** de como estruturar um `README.md` de hub dentro de `Documentation/` (ou de uma subpasta como `Documentation/Analise/`, `Documentation/Arquitetura/`, etc.).

Use este exemplo como base e adapte para o seu projeto.

| Campo | Valor |
| --- | --- |
| **Projeto** | `{PROJECT_NAME}` |
| **Versão** | `{DOCS_VERSION}` |
| **Data** | `{DATE}` |
| **Raiz** | `Documentation/` |

## Índice

- **Visão geral**: `Documentation/README_Vx.y.md`
- **Arquitetura**: `Documentation/Arquitetura/`
- **Análise**: `Documentation/Analise/`
- **Regras de Negócio**: `Documentation/Regras de Negocio/`
- **Versionamento**: `Documentation/Versionamento/CHANGELOG.md`

## Convenções

- **Caminhos**: links Markdown sempre **relativos ao ficheiro actual**.
- **Raiz canónica**: `Documentation/` (não usar `Docs/`/`docs/` em projetos novos; tratar como alias legado quando existir).
