| **FLUXO PRINCIPAL — Obtenção e Uso do Token** |  |  |  |
| --- | --- | --- | --- |
| 1\. API Central inicializa → TSerproTokenManager.Instance criado → verifica token cacheado no config.db.   2\. Se token cacheado válido (ExpiresAt > Now + 300s): carrega em memória, pronto para uso.   3\. Qualquer unit do M05 chama GetValidToken() antes de cada requisição SERPRO.   4\. GetValidToken() verifica IsExpiringSoon() → False: retorna token em memória (sem IO).   5\. Se IsExpiringSoon() = True: RequestNewToken() em background → token renovado transparentemente.   6\. Token utilizado como Bearer header em todas as chamadas: Authorization: Bearer {token}. |  |  |  |

| **FLUXOS DE EXCEÇÃO** |  |  |  |
| --- | --- | --- | --- |
| E1. HTTP 401 recebido do SERPRO (token expirado inesperadamente) → ForceRenew() · RequestNewToken() · retry 1× da chamada original com novo token · SERPRO\_TOKEN\_REFRESHED no audit\_log   E2. RequestNewToken() falha 3× (SERPRO indisponível) → SERPRO\_TOKEN\_RENEWAL\_FAILED no audit\_log · alerta ao administrador · GetValidToken() lança ESerproAuthException · chamadas SERPRO bloqueadas até resolução   E3. Certificado e-CNPJ vencido → mTLS falha no handshake · exceção TLS · SERPRO\_CERT\_EXPIRED no audit\_log · alerta crítico ao administrador (certificado precisa ser renovado junto ao ICP-Brasil)   E4. config.db indisponível ao persistir token → SetValue() falha silenciosamente · token válido em memória para a sessão atual · no próximo restart o token precisará ser renovado |  |  |  |

| **VALIDAÇÕES** |  |  |  |
| --- | --- | --- | --- |
| **Verificação** | **Condição** | **Resultado** | **Ação** |
| Token válido em memória | ExpiresAt &gt; Now + 300s | Retorna sem IO — zero latência adicional | — |
| Renovação preventiva | ExpiresAt ≤ Now + 300s | RequestNewToken() em background | BG |

| Persistência | Token salvo criptografado no config.db | Sobrevive a restart sem nova autenticação | App |

| // padrão.Parameters — SERPRO OAuth2   // GetValue('SERPRO', 'ClientID', s) → client\_id fornecido pelo SERPRO   // GetValue('SERPRO', 'ClientSecret', s) → client\_secret (criptografado)   // GetValue('SERPRO', 'CertPath', s) → caminho do .pfx do e-CNPJ   // GetValue('SERPRO', 'CertPassword', s) → senha do .pfx (criptografada)   // GetValue('SERPRO', 'TokenEndpoint', s) → URL de token OAuth2 SERPRO   // GetValue('SERPRO', 'RenewBeforeSec',s) → '300' (renovar 5 min antes)   // GetValue('SERPRO', 'AccessToken', s) → token em cache (criptografado)   // GetValue('SERPRO', 'TokenExpiresAt',s) → timestamp ISO8601 de expiração   // ── Unit: padrão.SerproAuth ────────────────────────────────────────────────   // Gerencia o ciclo de vida do access\_token OAuth2 com e-CNPJ A1/A3.   // Thread-safe: TMonitor para acesso concorrente ao token em memória.   unit padrão.SerproAuth;   interface   uses System.SysUtils, System.SyncObjs, System.DateUtils,   System.Net.HttpClient, System.Net.URLClient,   padrão.Parameters, padrão.AuditLog;   type   TSerproTokenManager = class   private   class var FInstance : TSerproTokenManager;   FAccessToken : string;   FExpiresAt : TDateTime;   FLock : TCriticalSection;   procedure RequestNewToken;   function IsExpiringSoon: Boolean;   public   constructor Create;   destructor Destroy; override;   class function Instance: TSerproTokenManager;   // Retorna token válido — renova se necessário   function GetValidToken: string;   // Forçar renovação imediata (chamado após HTTP 401)   procedure ForceRenew;   end;   implementation   class function TSerproTokenManager.Instance: TSerproTokenManager;   begin   if not Assigned(FInstance) then   FInstance := TSerproTokenManager.Create;   Result := FInstance;   end;   constructor TSerproTokenManager.Create;   var   sCachedToken : string;   sExpiresAt : string;   begin   inherited Create;   FLock := TCriticalSection.Create;   // Carregar token cacheado do config.db (persistido entre restarts)   GetValue('SERPRO', 'AccessToken', sCachedToken);   GetValue('SERPRO', 'TokenExpiresAt', sExpiresAt);   if not sCachedToken.IsEmpty and not sExpiresAt.IsEmpty then   begin   FAccessToken := DecryptSecret(sCachedToken);   FExpiresAt := ISO8601ToDateTime(sExpiresAt);   end;   end;   // ── Obter token válido (thread-safe) ─────────────────────────────────────────   function TSerproTokenManager.GetValidToken: string;   begin   FLock.Acquire;   try   // Verificar validade antes de servir   if FAccessToken.IsEmpty or IsExpiringSoon then   RequestNewToken;   Result := FAccessToken;   finally   FLock.Release;   end;   end;   // ── Verificar se token expira em breve ────────────────────────────────────────   function TSerproTokenManager.IsExpiringSoon: Boolean;   var   sRenewBefore: string;   iSecs : Integer;   begin   GetValue('SERPRO', 'RenewBeforeSec', sRenewBefore);   iSecs := StrToIntDef(sRenewBefore, 300);   Result := SecondsBetween(FExpiresAt, Now) <= iSecs;   end;   // ── Solicitar novo token ao SERPRO ────────────────────────────────────────────   procedure TSerproTokenManager.RequestNewToken;   var   oHttp : THTTPClient;   oResponse : IHTTPResponse;   sEndpoint : string;   sClientID : string;   sClientSecret : string;   sCertPath : string;   sCertPassword : string;   sBody : string;   oJSON : TJSONObject;   sNewToken : string;   iExpiresIn : Integer;   begin   GetValue('SERPRO', 'TokenEndpoint', sEndpoint);   GetValue('SERPRO', 'ClientID', sClientID);   GetValue('SERPRO', 'ClientSecret', sClientSecret);   GetValue('SERPRO', 'CertPath', sCertPath);   GetValue('SERPRO', 'CertPassword', sCertPassword);   oHttp := THTTPClient.Create;   try   // Configurar certificado e-CNPJ (mTLS)   oHttp.ClientCertificate(   sCertPath,   DecryptSecret(sCertPassword)   );   // Requisição OAuth2 client\_credentials   sBody :=   'grant\_type=client\_credentials' +   '&client\_id=' + sClientID +   '&client\_secret=' + DecryptSecret(sClientSecret);   oResponse := oHttp.Post(   sEndpoint,   TStringStream.Create(sBody),   nil,   \[TNameValuePair.Create('Content-Type',   'application/x-www-form-urlencoded')\]   );   if oResponse.StatusCode <> 200 then   raise Exception.CreateFmt(   'SERPRO token endpoint retornou HTTP %d: %s',   \[oResponse.StatusCode, oResponse.ContentAsString\]);   // Parsear resposta JSON   oJSON := TJSONObject.ParseJSONValue(   oResponse.ContentAsString) as TJSONObject;   sNewToken := oJSON.GetValue<string>('access\_token');   iExpiresIn := oJSON.GetValue<Integer>('expires\_in'); // segundos (3600)   FAccessToken := sNewToken;   FExpiresAt := IncSecond(Now, iExpiresIn);   // Persistir no config.db (criptografado) para sobreviver a restarts   SetValue('SERPRO', 'AccessToken',   EncryptSecret(sNewToken));   SetValue('SERPRO', 'TokenExpiresAt',   DateTimeToISO8601(FExpiresAt));   TAuditLog.Write(TGuid.Empty, 'SERPRO\_TOKEN\_REFRESHED',   'serpro', 'token', '',   Format('{"expires\_in":%d}', \[iExpiresIn\]));   finally   oHttp.Free;   end;   end;   // ── Forçar renovação após HTTP 401 ────────────────────────────────────────────   procedure TSerproTokenManager.ForceRenew;   begin   FLock.Acquire;   try   FAccessToken := ''; | **LGPD / NOTAS** |  |  |  |
| --- | --- | --- | --- |
| (x) Toca dados sensíveis do escritório (certificado e-CNPJ): o certificado representa a identidade jurídica do escritório. Se comprometido, permite assinar documentos em nome do escritório junto à RFB. O armazenamento com AES-256 no config.db e a não-exposição em logs são medidas de segurança obrigatórias.   O padrão de renovação preventiva (300s antes) foi escolhido para evitar falhas em chamadas longas: se uma operação Integra Contador demorar 60 segundos e o token expirar durante a execução, o 401 poderia ser recebido no meio da operação. A renovação antecipada previne esse cenário.   Atenção operacional: o certificado e-CNPJ A1 tem validade de 1 ano. O padrão deve alertar o administrador com 30 dias de antecedência sobre o vencimento — verificação implementada junto ao monitoramento de certidões (RN-M05-005) como boa prática. |  |  |  |
